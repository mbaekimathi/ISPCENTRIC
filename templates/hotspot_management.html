Hotspot network (guest SSID)

Router: {{ router_name }} — Broadcast by client CPEs as a separate SSID; does not change the client’s own WiFi.

How it works: Set the SSID and guest security mode here. For captive portal flow, enable Captive portal mode and provide your portal URL; this forces open guest WiFi and users should be redirected to your hotspot website to select a plan before internet access.

Used only when captive portal mode is enabled.

Built-in portal URL: {{ default_portal_url }}

Portal Only captures plan requests and redirects users; MikroTik Enforced auto-grants/blocks access via RouterOS hotspot bindings.

If set, use WPA2 for the hotspot. If empty, the hotspot can be open. Client’s own WiFi settings are not changed.

No changes are made to the MikroTik or to any customer’s existing WiFi. Use these credentials to configure the guest/hotspot SSID on each client’s router (second broadcast only). Users who connect to this SSID get internet; the client’s own network is unaffected.
